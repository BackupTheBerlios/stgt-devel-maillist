Yep. The SSC metadata (header for each tape block) contains a
structure with a 'block type' , size of original data & size of data
stored.

All accounted for. Just no code for compression or encryption - as yet..

enum {
        BLK_NOOP,
        BLK_UNCOMPRESS_DATA,
        BLK_COMPRESSED_DATA,
        BLK_ENCRYPTED_DATA,
        BLK_FILEMARK,
        BLK_BOT,
        BLK_EOD,
};
struct blk_header {
        uint32_t blk_type;
        uint32_t blk_size;
        uint32_t disk_blk_size;
        loff_t blk_number;
        loff_t prev_blk;
        loff_t curr_blk;
        loff_t next_blk;
};
